# UX / UI Guidelines

## 1. Философия

- **Аудитория**: Учителя начальных классов.
- **Принципы**:
  - **Минимализм**: Ничего лишнего, фокус на форме.
  - **Прозрачность**: Пользователь всегда видит прогресс (0-100%).
  - **Скорость**: Результат должен быть доступен максимально быстро.

---

## 2. Цветовая Палитра

- **Primary**: Indigo-600 (`#4f46e5`) — основной акцент.
- **Background**: White (`#ffffff`) / Slate-50 (`#f8fafc`).
- **Text**: Slate-900 (основной), Slate-500 (второстепенный).
- **Error**: Rose-500 (`#f43f5e`).
- **Success**: Emerald-500 (`#10b981`).

---

## 3. Экраны

### 3.1 Главная (`/`)

1.  **Hero**: Заголовок + подзаголовок.
2.  **Form**:
    - **Предмет**: Select (Математика / Русский).
    - **Класс**: Select (1-4).
    - **Тема**: Input (с валидацией длины).
    - **CTA**: Кнопка "Сгенерировать".
3.  **Loading State**:
    - Прогресс-бар с текстовым пояснением этапа (например, "Создаем задания...", "Генерируем PDF...").
    - Блокировка повторной отправки.

### 3.2 Страница Листа (`/worksheet/:id`)

1.  **Header**: Навигация + Кнопка "Скачать PDF".
2.  **Preview**: Визуальное отображение листа (HTML/CSS), максимально похожее на PDF.
    - Блоки заданий.
    - Тест.
    - Ответы (скрыты под спойлером или внизу).
3.  **Actions**:
    - "Перегенерировать" (с сохранением параметров).
    - "Создать новый".

---

## 4. Компоненты (shadcn/ui)

- **Button**: Default, Outline, Ghost.
- **Input / Select**: Стандартные стили shadcn.
- **Card**: Для группировки блоков заданий.
- **Progress**: Для отображения статуса генерации.
- **Toast**: Для уведомлений об ошибках.

---

## 5. Доступность (A11y)

- Все поля формы имеют `label`.
- Контрастность текста соответствует WCAG AA.
- Навигация с клавиатуры.
- ARIA-атрибуты для динамических элементов.
